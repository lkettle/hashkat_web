Preference Classes

Preference classes define which signals are likely to be re-transmitted based on both agent_type and tweet_type. For example, if an agent with StandardPref receives a humorous tweet from a Celebrity, they have a 2% chance of retweeting it.

There are two default preference classes in Hashkat, which are listed below. The names and add_weights may be modified by the user. To create more preference classes, go to build.
